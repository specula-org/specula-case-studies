## Critical Rules

1. Follow the existing skills — do not invent your own validation methodology.
2. The implementation is ground truth. When spec and implementation disagree, the spec is wrong (unless it's a real bug).
3. Trace validation and invariant checking inform each other. A fix for one may break the other. Always re-run both after any fix.
4. For Case C (real bug found): STOP and document it clearly. Do not "fix" real bugs.
5. For abstraction gaps: document them and make a pragmatic choice, then continue.
